📘 Pandas Topic 3: Exploring a DataFrame
What is DataFrame Exploration?

DataFrame Exploration is the process of examining and understanding a dataset before performing any analysis or building a Machine Learning model.

It helps us understand the structure, size, data types, statistics, and overall quality of the data.

Data Exploration is always the first step in Data Analysis, Data Science, and Machine Learning.

Definition

DataFrame Exploration is the process of inspecting a dataset to understand its structure, contents, dimensions, data types, and statistical information before performing further analysis.

Why Do We Explore a DataFrame?

We explore a DataFrame to:

Understand the dataset.
Check the number of rows and columns.
View the first and last records.
Identify column names.
Check data types.
Get summary statistics.
Detect missing values.
Understand the overall quality of the data.
Common DataFrame Exploration Functions
1. head()
What is head()?

The head() function displays the first five rows of a DataFrame by default.

It gives a quick overview of how the dataset looks.

Why is it used?
To preview the dataset.
To check whether the data has been loaded correctly.
To understand the structure of the table.
Real-World Example

Imagine you receive a 500-page book.

Instead of reading the whole book immediately, you first read the first few pages to understand what it is about.

head() works in the same way.

2. head(n)
What is head(n)?

head(n) displays the first n rows of the DataFrame.

Instead of showing the default five rows, you can choose exactly how many rows you want to see.

Example
head(3) → Shows the first 3 rows.
head(10) → Shows the first 10 rows.
3. tail()
What is tail()?

The tail() function displays the last five rows of a DataFrame by default.

It helps you see how the dataset ends.

Why is it used?
To inspect the ending records.
To verify the last entries in the dataset.
To check whether the data is complete.
Real-World Example

Suppose you finish reading a book.

Before buying it, you quickly read the last few pages to see how it ends.

That is exactly what tail() does.

4. tail(n)
What is tail(n)?

tail(n) displays the last n rows of the DataFrame.

Example
tail(2) → Shows the last 2 rows.
tail(10) → Shows the last 10 rows.
5. Shape
What is shape?

The shape property returns the size of the DataFrame.

It tells us:

Number of rows.
Number of columns.

The output is always in the form:

(rows, columns)
Why is it used?

It helps us understand how large the dataset is.

Example
(7, 5)

means:

7 rows
5 columns
Real-World Example

Imagine a classroom has:

7 students
5 subjects

The classroom data has the shape:

(7,5)
6. Columns
What is columns?

The columns property returns the names of all columns in a DataFrame.

It helps us understand what information is available.

Why is it used?
To view all column names.
To verify the dataset structure.
To select columns during analysis.
Example
Name
Age
City
Marks
Grade
Real-World Example

Think of a school register.

The headings at the top such as:

Name
Age
Marks
Grade

are the column names.

7. Data Types (dtypes)
What is dtypes?

The dtypes property shows the data type of each column.

Different columns may store different kinds of data.

Common Data Types
int64 → Integer numbers
float64 → Decimal numbers
object → Text (strings)
bool → True or False values
Why is it used?

It helps us know whether each column contains:

Numbers
Text
Boolean values
Real-World Example

A teacher checks the register and separates:

Numerical information (Age, Marks)
Text information (Name, City)
8. describe()
What is describe()?

The describe() function provides a statistical summary of all numerical columns in a DataFrame.

It quickly summarizes the dataset without performing manual calculations.

It provides:
Count
Mean
Standard Deviation
Minimum value
25th Percentile
50th Percentile (Median)
75th Percentile
Maximum value
Explanation of Statistics
Count

Shows how many values are present.

Mean

Returns the average value.

Standard Deviation (Std)

Shows how much the data is spread out from the average.

A small standard deviation means the values are close together.

A large standard deviation means the values are more spread out.

Minimum (Min)

Shows the smallest value.

Maximum (Max)

Shows the largest value.

25%

Shows the value below which 25% of the data lies.

50%

Shows the middle value (Median).

75%

Shows the value below which 75% of the data lies.

Why is describe() important?

It gives a complete statistical overview of the dataset in a single step.

Real-World Example

A teacher wants to know:

Average marks
Highest marks
Lowest marks

Instead of calculating everything manually, the teacher gets all the information instantly.

9. info()
What is info()?

The info() function provides complete information about the DataFrame.

It shows:

Number of rows.
Number of columns.
Column names.
Non-null values.
Data types.
Memory usage.
Why is it important?

It helps identify:

Missing values.
Incorrect data types.
Overall dataset information.
Real-World Example

Imagine a school administrator checking a student register.

They verify:

How many students are registered.
Whether any student information is missing.
Which columns contain numbers or text.

This is exactly what info() does.

Key Points to Remember
Data exploration is always the first step before analysis.
head() shows the first rows.
tail() shows the last rows.
shape shows the dataset size.
columns displays column names.
dtypes displays data types.
describe() provides statistical information.
info() provides complete dataset information, including missing values.

🎯 Real-Life Example

Imagine a company gives a Data Analyst a dataset containing 100,000 customer records.

Before performing any analysis, the analyst first:

Views the first few records using head().
Checks the last few records using tail().
Finds the dataset size using shape.
Reviews all column names using columns.
Checks data types using dtypes.
Gets summary statistics using describe().
Detects missing values using info().

Only after understanding the dataset does the analyst begin the actual analysis.

⭐ Exam Definition
