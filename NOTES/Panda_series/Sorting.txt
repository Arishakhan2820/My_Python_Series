📘 Pandas Topic 9: Sorting Data
What is Sorting?

Sorting is the process of arranging data in a specific order. In Pandas, sorting helps organize rows based on the values of one or more columns.

In simple words,

Sorting means arranging data from smallest to largest, largest to smallest, or alphabetically so that it becomes easier to read and analyze.

Definition

Sorting in Pandas is the process of organizing rows based on the values of one or more columns using the sort_values() function.

Sorting makes data easier to understand, compare, and analyze.

Why Do We Use Sorting?

Sorting is used to:

Organize data in a meaningful order.
Find the highest or lowest values quickly.
Create professional reports.
Compare records easily.
Improve data analysis.
Prepare data for visualization and Machine Learning.
Types of Sorting
1. Ascending Sorting

Ascending sorting arranges data from:

Smallest → Largest
A → Z (for text)

This is the default sorting order in Pandas.

Example

Marks:

85, 90, 78, 92, 88

After ascending sorting:

78, 85, 88, 90, 92

Real-Life Example

A teacher arranges students from lowest marks to highest marks.

2. Descending Sorting

Descending sorting arranges data from:

Largest → Smallest
Z → A (for text)
Example

Marks:

85, 90, 78, 92, 88

After descending sorting:

92, 90, 88, 85, 78

Real-Life Example

A school displays the top-performing students on the notice board, so students with the highest marks appear first.

Sorting by Multiple Columns

Sometimes one column is not enough.

Pandas allows sorting using multiple columns.

For example:

First sort by Age.
If two students have the same age, then sort them by Marks.

This is called Multi-Level Sorting.

Real-Life Example

A company wants to arrange employees:

By Department.
If employees belong to the same department, arrange them by Salary.

This produces a more organized report.

Resetting the Index

After sorting, the row order changes, but the original index numbers remain the same.

To create a clean and organized DataFrame, Pandas provides the reset_index() function.

This creates new row numbers starting from 0.

Why Use drop=True?

When resetting the index:

Without drop=True, the old index is added as a new column.
With drop=True, the old index is removed completely.

This keeps the DataFrame clean.
