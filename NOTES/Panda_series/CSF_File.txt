📘 Pandas Topic 10: Reading & Writing CSV Files (Real-World Data Handling)

What is a CSV File?

CSV stands for Comma Separated Values.

A CSV file is a simple text file used to store data in rows and columns, where each value is separated by a comma.

It is one of the most commonly used file formats for storing and sharing data because it is simple, lightweight, and supported by almost every software, including Excel, Google Sheets, SQL, and Python.

Definition

A CSV (Comma Separated Values) file is a file format used to store tabular data in rows and columns. In Pandas, CSV files can be read into a DataFrame using read_csv() and written (saved) from a DataFrame using to_csv().

Why Do We Use CSV Files?

CSV files are used because they:

Store large amounts of tabular data.
Are easy to read and write.
Can be shared between different software.
Are supported by Excel, Google Sheets, SQL, Python, and many other tools.
Are widely used in Data Analysis and Machine Learning projects.
Reading a CSV File

Reading a CSV file means loading data from a CSV file into a Pandas DataFrame.

After reading the file, the data becomes easy to analyze, filter, sort, clean, and visualize.

Real-Life Example

A company sends you a file named sales_2025.csv containing sales data.

Before analyzing the sales, you first load the CSV file into Pandas using read_csv().

Writing (Saving) a CSV File

Writing a CSV file means saving a Pandas DataFrame into a CSV file.

This is useful after cleaning, modifying, or analyzing the data.

Real-Life Example

You cleaned customer data by removing duplicates and filling missing values.

Now you save the cleaned dataset as cleaned_customers.csv so your team can use it.

Why Use index=False?

By default, Pandas saves row index numbers along with the data.

Using index=False prevents the row numbers from being saved.

This creates a cleaner CSV file containing only the actual data.

Example

Without index=False

Index	Name	Marks
0	Ali	85
1	Sara	90

With index=False

Name	Marks
Ali	85
Sara	90
Exploring a CSV File

Whenever you receive a new CSV file, you should explore it before performing any analysis.

The most common checks are:

1. Shape

Shows the total number of rows and columns.

It helps you understand the size of the dataset.

2. Head

Displays the first few rows of the dataset.

It helps you quickly understand what the data looks like.

3. Data Types

Shows the data type of each column.

Examples:

Text (object)
Integer (int64)
Decimal (float64)
Boolean (bool)

Knowing data types helps perform the correct calculations.

4. Missing Values

Checking missing values helps identify incomplete data.

Missing values should usually be cleaned before analysis or Machine Learning.

5. Statistical Summary

The statistical summary provides useful information about numerical columns, such as:

Count
Average (Mean)
Minimum value
Maximum value
Standard Deviation
Percentiles

This gives a quick overview of the dataset.

Standard Workflow for a New CSV File

Whenever you receive a new dataset, follow this order:

Step 1

Read the CSV file.

Step 2

Check the dataset size using Shape.

Step 3

View the first few rows using Head.

Step 4

Check the data types.

Step 5

Find missing values.

Step 6

View the statistical summary.

Step 7

Clean or modify the data if necessary.

Step 8

Save the cleaned dataset as a new CSV file.

This is the standard workflow followed by Data Analysts and Data Scientists.
