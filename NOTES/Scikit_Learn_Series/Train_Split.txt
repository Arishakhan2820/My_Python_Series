📘 Topic 1 — Train-Test Split (Machine Learning)

What is Train-Test Split?

Train-Test Split is a Machine Learning technique used to divide a dataset into two parts:

Training Data – Used to teach (train) the machine learning model.
Testing Data – Used to evaluate (test) how well the trained model performs on new, unseen data.

In simple words:

Train-Test Split means dividing a dataset into two parts: one part is used for learning, and the other part is used to test how well the model has learned.

Why Do We Use Train-Test Split?

If we train and test a model using the same data, the model may simply memorize the answers instead of learning patterns. This can lead to inaccurate performance in real-world situations.

Using separate training and testing data helps us evaluate how well the model performs on new data.

How Does Train-Test Split Work?

The dataset is divided into two parts:

1. Training Set

The training set is used to teach the model.

The model learns patterns, relationships, and trends from this data.

2. Testing Set

The testing set is not shown during training.

After training is complete, this unseen data is used to check the model's performance.

Common Split Ratios

There is no fixed rule, but these are the most commonly used ratios:

Training Data	Testing Data	Usage
80%	20%	Most common choice
70%	30%	Medium-sized datasets
90%	10%	Very large datasets
75%	25%	Sometimes used in practice

The 80:20 split is the most popular because it provides enough data for training while keeping sufficient data for testing.

What are Features (X)?

Features (X) are the input values given to the machine learning model.

They contain the information the model uses to make predictions.

Example:

Suppose we want to predict a student's marks.

Features may include:

Study Hours
Attendance
Assignment Score

These input values are represented by X.

What is Target (y)?

The Target (y) is the output or answer that the model tries to predict.

Example:

If the model predicts exam marks:

Study Hours (X) → Input
Exam Marks (y) → Output
What is train_test_split()?

train_test_split() is a function from Scikit-learn that automatically divides the dataset into training and testing sets.

It saves time because we do not need to split the data manually.

Important Parameters
1. test_size

test_size specifies how much data should be used for testing.

Example:

test_size = 0.2

This means:

80% Training Data
20% Testing Data
2. random_state

random_state controls how the data is shuffled before splitting.

Example:

random_state = 42

Using the same value ensures that the split remains the same every time the code runs.

This makes results reproducible.

Why is Random Shuffling Important?

If the data is already ordered, splitting without shuffling may produce biased results.

Random shuffling ensures that both training and testing datasets contain a good mix of samples.

Real-World Example

Imagine a teacher has 100 practice questions.

Instead of practicing all 100 questions and then taking the same test, the teacher does this:

80 questions → Practice (Training)
20 questions → Final Test (Testing)

If the student performs well on the 20 unseen questions, it means they have truly learned the concepts.

Machine Learning works in exactly the same way.

Advantages
Easy to use.
Prevents testing on training data.
Gives a fair evaluation of the model.
Helps detect overfitting.
Improves model reliability.
Standard practice in Machine Learning.
Limitations
Small datasets may not provide enough training or testing data.
Different random splits may produce slightly different results.
Not always suitable for very small datasets (Cross Validation is often preferred).
