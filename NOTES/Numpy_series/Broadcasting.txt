📘 Topic: Broadcasting in NumPy
What is Broadcasting?

Broadcasting is a smart feature in NumPy that automatically expands a smaller array (or a single value) to match the shape of a larger array so that mathematical operations can be performed without using loops.

In simple words:

Broadcasting means NumPy automatically applies a smaller array or a single value to every required element of a larger array during an operation.

Note: NumPy does not actually create extra copies in memory. It only behaves as if it did, making operations faster and more memory-efficient.

Why is Broadcasting Used?

Broadcasting is used to make numerical operations:

Faster
Simpler
Cleaner
More memory-efficient
Easier to write and understand

Instead of writing loops to process every element one by one, NumPy performs the operation automatically.

How Does Broadcasting Work?

When two arrays have different sizes or shapes, NumPy checks whether they are compatible for broadcasting.

If they are compatible, NumPy automatically expands the smaller array (or scalar) so it behaves like the larger array during the operation.

This allows element-wise operations such as addition, subtraction, multiplication, and division without manually matching the array sizes.

Why is Broadcasting Important?

Broadcasting is one of the most powerful features of NumPy because it:

Eliminates the need for manual loops.
Makes code shorter and easier to read.
Improves execution speed.
Uses memory efficiently.
Simplifies operations on large datasets.
