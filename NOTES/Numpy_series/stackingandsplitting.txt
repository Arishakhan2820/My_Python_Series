📘 Topic 4: Stacking & Splitting in NumPy

What is Stacking?

Stacking is the process of joining two or more arrays together to create a larger array.

Instead of creating a new array manually, NumPy provides built-in functions that combine arrays quickly and efficiently.

In simple words:

Stacking means combining multiple arrays into one larger array.

Why is Stacking Used?

Stacking is used to:

Combine multiple datasets into one.
Organize data in a structured way.
Prepare data before analysis or machine learning.
Increase the size of a dataset.
Make data easier to process.

Instead of handling many small arrays separately, stacking allows us to work with a single combined array.

How Does Stacking Work?

NumPy takes two or more arrays and joins them together in a specific direction.

There are different ways to stack arrays depending on where you want to place them.

You can join arrays:

Vertically (top to bottom)
Horizontally (left to right)
Along a specific axis
Types of Stacking

NumPy provides three common ways to stack arrays.

1. Vertical Stacking (vstack)
What is Vertical Stacking?

Vertical stacking joins arrays from top to bottom.

One array is placed below another array.

Think of it like adding more rows.

Real-Life Example

Imagine you have two notebooks.

Notebook 1 contains:

Student A
Student B

Notebook 2 contains:

Student C
Student D

If you combine both notebooks into one list by placing the second notebook below the first, you get one complete list of students.

This is exactly how Vertical Stacking works.

AI Example

Suppose a hospital collects patient data from two different departments.

Department A has 500 patient records.
Department B has 500 patient records.

Before training an AI model, both datasets are combined into one large dataset using vertical stacking.

2. Horizontal Stacking (hstack)
What is Horizontal Stacking?

Horizontal stacking joins arrays side by side.

Instead of adding more rows, it adds more columns.

Real-Life Example

Imagine a school has two separate files.

The first file contains:

Student Name
Roll Number

The second file contains:

Math Marks
Science Marks

If both files are placed side by side, each student's complete information is available in one table.

This is Horizontal Stacking.

AI Example

Suppose an AI model is predicting house prices.

One dataset contains:

House Size
Number of Bedrooms

Another dataset contains:

Location Score
Distance from City

Before training the model, both datasets are combined side by side using horizontal stacking.

Now every house has all its features in one dataset.

3. Concatenate
What is Concatenate?

Concatenate is a flexible function that joins arrays along a chosen direction (called an axis).

Unlike vertical and horizontal stacking, concatenate allows you to decide how the arrays should be combined.

It is one of the most commonly used functions for joining arrays.

Real-Life Example

Imagine you have several chapters of a book stored separately.

Instead of reading them individually, you combine them into one complete book.

Concatenate works in a similar way by joining separate arrays into one larger array.

AI Example

In Machine Learning, data may come from different sources such as customer information, sales records, and website activity.

These datasets are combined into one complete dataset before training the model.

What is Splitting?

Splitting is the opposite of stacking.

Instead of joining arrays together, splitting divides one large array into multiple smaller arrays.

In simple words:

Splitting means breaking one large array into smaller parts.

Why is Splitting Used?

Splitting is used to:

Divide large datasets into smaller parts.
Process data in batches.
Organize data.
Separate training and testing data.
Improve memory usage.
How Does Splitting Work?

NumPy divides a large array into multiple smaller arrays.

The split can be:

Equal-sized parts
Custom-sized parts
Types of Splitting
Equal Splitting

The array is divided into equal-sized sections.

Each part contains the same number of elements.

Real-Life Example

Imagine a teacher has 30 chocolates.

The teacher wants to distribute them equally among 5 students.

Each student receives the same number of chocolates.

This is equal splitting.

AI Example

Suppose an AI model needs to process 10,000 images.

Instead of processing all images at once, the images are divided into smaller groups called batches.

Each batch is processed one at a time.

This makes training faster and uses less memory.

Custom Splitting

Sometimes equal parts are not needed.

Instead, we decide exactly where the array should be divided.

Real-Life Example

Imagine a pizza.

Instead of cutting it into equal slices,

you decide to make:

2 slices for children
4 slices for adults
Remaining slices for guests

The cuts are made according to your choice.

This is custom splitting.

AI Example

Suppose customer data contains:

Personal Information
Purchase History
Payment Details

Instead of keeping everything together, the dataset is divided into separate sections for easier analysis.

Difference Between Stacking and Splitting
Stacking	Splitting
Combines arrays into one larger array.	Divides one array into smaller arrays.
Creates a larger dataset.	Creates multiple smaller datasets.
Used before processing data.	Used when dividing data into parts.
Increases the size of data.	Reduces the size of each part.
Why are Stacking and Splitting Important?

Both operations are essential because they help organize data efficiently.

They make it easier to:

Prepare datasets
Combine information
Divide large data
Improve memory usage
Process data faster
Train AI models efficiently

Without stacking and splitting, managing large datasets would be much more difficult.

AI & Machine Learning Connection

Before an AI model is trained, data usually comes from multiple sources.

For example:

Customer information
Purchase history
Website activity
Product details

These datasets are first stacked together to create one complete dataset.

Later, the combined dataset is split into training, validation, and testing sets.

The AI model learns from the training data and is evaluated using the testing data.

This is one of the most common workflows in Machine Learning.

Common Beginner Mistakes

❌ Thinking stacking and concatenation are exactly the same.

❌ Confusing vertical stacking with horizontal stacking.

❌ Splitting an array into unequal parts without understanding the required size.

❌ Forgetting that arrays must have compatible shapes before stacking.
