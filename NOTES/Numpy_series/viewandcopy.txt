📘 Topic 5: View & Copy in NumPy

What is a View?

A View is a new array that shares the same memory as the original array.

It does not create a new copy of the data. Instead, it simply provides another way to access the original array.

In simple words:

A View is another window to look at the same original array.

Since both arrays use the same memory, changing one will also change the other.

Why is a View Used?

A View is used because it:

Saves memory.
Is faster than creating a copy.
Avoids unnecessary duplication of data.
Is useful when we only want another reference to the same data.
How Does a View Work?

When a View is created, NumPy does not make a new array.

Instead, both the original array and the View point to the same memory location.

If you modify the View, the original array also changes because both are sharing the same data.

Real-Life Example (View)

Imagine you are standing outside your house and looking through a window.

The window is not another room.

It simply lets you see the same room.

If someone changes the furniture inside the room, you immediately see the change through the window.

The room and the window are connected.

A View works in exactly the same way.

AI Example (View)

Suppose an AI engineer is analyzing a very large dataset containing 1 million customer records.

Instead of creating another full copy of the dataset, they create a View to work with only a selected portion of the data.

This saves memory and improves performance because both arrays share the same data.

What is a Copy?

A Copy is a completely new array with its own separate memory.

It contains the same values as the original array, but it is completely independent.

In simple words:

A Copy is a duplicate of the original array.

Changing the Copy does not affect the original array.

Why is a Copy Used?

A Copy is used when we want to:

Protect the original data.
Experiment safely.
Modify data without changing the source.
Create an independent version of an array.
How Does a Copy Work?

When a Copy is created, NumPy creates a new block of memory.

The original array and the Copy are stored separately.

Because they have different memory locations, changing one does not affect the other.

Real-Life Example (Copy)

Imagine you have a photocopy of a book.

You can highlight, write, or draw on the photocopy.

The original book remains exactly the same.

A Copy in NumPy behaves in the same way.

AI Example (Copy)

Suppose a data scientist is cleaning a medical dataset before training an AI model.

Instead of modifying the original patient records, they first create a Copy.

They clean and transform the copied dataset while keeping the original data safe.

This is a common practice in Machine Learning.

What is .base?

The .base attribute helps us check whether an array shares memory with another array.

If .base refers to the original array, it is a View.
If .base is None (or not the original array), it is an independent Copy.

Common Beginner Mistakes

❌ Thinking that slicing always creates a new array.

❌ Forgetting that a View shares memory with the original array.

❌ Modifying a View and expecting the original array to stay unchanged.

❌ Using a Copy when a View would be more memory-efficient.

Interview Notes ⭐
Question:

What is the difference between View and Copy in NumPy?

Answer:

A View shares memory with the original array, so changes made to the View also affect the original array.

A Copy creates a completely independent array with separate memory, so changes made to the Copy do not affect the original array.

Key Points to Remember
A View shares memory with the original array.
A Copy has separate memory.
Slicing usually creates a View.
The .copy() method creates an independent Copy.
View is faster and memory-efficient.
Copy is safer when modifying data.
.base helps identify whether an array is a View.

Exam Summary ⭐

View and Copy are important NumPy concepts used to manage array memory efficiently. A View shares the same memory as the original array, making it faster and more memory-efficient, while a Copy creates a completely independent array that protects the original data from accidental changes.
