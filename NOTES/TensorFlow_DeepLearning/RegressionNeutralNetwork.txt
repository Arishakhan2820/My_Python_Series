TOPIC 4 — Regression Neural Network:

 What is Regression?

Regression is a type of Machine Learning and Deep Learning problem where the goal is to predict a continuous numerical value instead of selecting a category.

Unlike classification, regression predicts numbers.

In Simple Words

Regression = Predicting a numerical value based on input data.

 Real-World Example:

Imagine you are a real estate agent.

A customer asks:

"How much will this house cost?"

You look at:

Number of rooms
House area
House age
Location

Then you estimate the house price.

A Regression Neural Network does exactly the same thing—it learns from past house data and predicts the price of a new house.

 Project Goal:

The goal of this project is to build a Regression Neural Network that predicts the price of a house.

The prediction is based on four features:

Number of Rooms
House Area (Square Feet)
House Age
Location Quality

📊 Dataset

A dataset is a collection of information.

Each house contains:

Input Features

These are the values used to make predictions.

Number of Rooms
Area (Square Feet)
Age of House
Location Rating
Output (Target)

The output is the House Price.

Unlike classification, the output is not Pass/Fail.

It is a continuous number such as:

Rs. 2,500,000
Rs. 4,800,000
Rs. 7,200,000

 Real-World Example:

Suppose two houses have the following details:

Rooms	Area	Age	Location	Price
3	1200 sqft	5 years	Prime	Rs. 4.2 Million
5	2500 sqft	2 years	Prime	Rs. 7.8 Million

The neural network studies many such examples and learns how these features affect the house price.

 Splitting the Dataset:

The dataset is divided into two parts.

Training Data

Used to teach the neural network.

Usually around 80% of the data.

Testing Data

Used to evaluate how well the model predicts prices for houses it has never seen before.

Usually around 20% of the data.

Real-World Example

Imagine solving 80 practice questions before an exam.

The remaining 20 questions are completely new.

If you solve them correctly, it shows that you truly understood the topic.

Testing data works the same way.

 Data Scaling (Standardization):

The values of different features have very different ranges.

Example:

Rooms:

1 to 5

Area:

500 to 3000

Location:

1 to 10

Price:

Millions of Rupees

If these values are used directly, the model may learn slowly.

Scaling converts the values into a similar range.

In this project:

Input features (X) are scaled.
Target values (House Prices) are also scaled.

After prediction, the prices are converted back to their original values.

Why is Scaling Important?
Faster learning
Better accuracy
Stable training
Easier optimization
Real-World Example

Imagine measuring one distance in meters and another in kilometers.

Comparing them directly is difficult.

Scaling converts everything into the same measurement system.

 Building the Regression Neural Network

The neural network contains several layers.

1. Input Layer

The Input Layer receives four features:

Rooms
Area
Age
Location

It simply forwards this information to the hidden layers.

Real-World Example

Like filling out a property information form before calculating the price.

2. Hidden Layers

Hidden Layers learn relationships between house features and house prices.

The model uses several hidden layers with many neurons.

Each hidden layer discovers more complex patterns.

Real-World Example

Imagine several experienced property experts discussing a house.

One checks:

Rooms

Another checks:

Area

Another checks:

Location

Together they estimate the price.

The hidden layers work similarly.

 Neurons:

A neuron is the smallest learning unit inside a neural network.

Each neuron:

Receives information
Performs calculations
Sends useful information to the next neuron

Thousands of neurons work together to make predictions.

 ReLU Activation Function:

The hidden layers use the ReLU (Rectified Linear Unit) activation function.

ReLU:

Converts negative values into zero.
Keeps positive values unchanged.
Why ReLU?
Faster learning
Simple calculations
Best choice for hidden layers
Real-World Example

Imagine a company only accepts profitable projects.

Loss-making projects are ignored.

ReLU works similarly.

 Dropout Layer:

Dropout prevents Overfitting.

During training, it randomly turns off some neurons.

This forces the network to learn better patterns instead of memorizing the data.

What is Overfitting?

Overfitting happens when the model memorizes training data instead of learning general rules.

Such a model performs poorly on new data.

Real-World Example

A student memorizes previous exam papers.

When the exam changes, the student struggles.

A student who understands concepts performs well everywhere.

Dropout encourages concept learning.

 Output Layer:

The output layer predicts the House Price.

Unlike classification problems, regression predicts any numerical value.

Therefore, the output layer uses the Linear Activation Function.

 Linear Activation Function:

Linear activation does not limit the output.

The predicted value can be:

Rs. 2 Million
Rs. 4 Million
Rs. 8 Million

There is no upper or lower limit.

Why Use Linear?

House prices can be any value.

They are not limited between 0 and 1.

Therefore, Linear activation is the correct choice.

Real-World Example

Imagine estimating the value of a luxury house.

Its price could be Rs. 20 Million or even Rs. 100 Million.

A fixed range would not work.

 Model Compilation:

Before training starts, the model must be compiled.

Compilation tells TensorFlow:

How to learn
How to measure mistakes
Which optimizer to use

🚀 Optimizer (Adam):

The optimizer adjusts the neural network's weights after every training step.

This project uses the Adam Optimizer.

Why Adam?
Fast learning
Stable performance
Automatically adjusts learning
Real-World Example

Imagine a student correcting mistakes after every test.

The student gradually improves.

Adam helps the neural network improve in the same way.

 Loss Function (Mean Squared Error - MSE)

The model uses Mean Squared Error (MSE).

MSE measures how far the predicted value is from the actual value.

Large errors receive a much larger penalty.

Why Use MSE?

Because regression predicts numbers.

The model should minimize prediction errors.

Smaller MSE means better predictions.

Real-World Example

Actual house price:

Rs. 5,000,000

Predicted:

Rs. 4,950,000

Small error → Small loss

Actual:

Rs. 5,000,000

Predicted:

Rs. 2,000,000

Huge error → Large loss

 Mean Absolute Error (MAE)

MAE measures the average absolute difference between predicted and actual values.

Unlike MSE, MAE treats all errors equally.

Why MAE?

It tells us, on average, how many rupees the prediction is away from the actual price.

Real-World Example

If MAE is:

Rs. 80,000

It means the model is usually off by about Rs. 80,000.

 Model Training:

Training is the process where the neural network learns from the training data.

During training:

It studies all houses.
Finds relationships.
Adjusts weights.
Improves after every epoch.

🔄 Epoch

An Epoch means the model has processed the entire training dataset once.

Example:

100 epochs mean the model studies the complete dataset 100 times.

More epochs usually improve learning, but too many may cause overfitting.

📦 Batch Size

Instead of processing all houses together, the model processes small groups.

Example:

Batch Size = 32

The model learns from 32 houses at one time.

Benefits
Faster training
Lower memory usage
Stable learning

 Early Stopping

Early Stopping automatically stops training when the model is no longer improving.

This prevents unnecessary training and reduces overfitting.

Why Use Early Stopping?
Saves time
Prevents overfitting
Keeps the best-performing model
Real-World Example

Imagine studying for an exam.

If your practice test scores stop improving even after many days, studying in the same way may no longer help.

It is better to stop or change your strategy.

Early Stopping works similarly.

📉 Reduce Learning Rate on Plateau

Sometimes the model stops improving because the learning rate is too high.

Reduce Learning Rate automatically lowers the learning rate when progress slows down.

Why Use It?
Helps the model fine-tune its learning.
Improves final accuracy.
Prevents the optimizer from skipping the best solution.
Real-World Example

Imagine riding a bicycle downhill.

At first you move quickly.

As you approach your destination, you slow down to stop accurately.

Reducing the learning rate works in the same way.

📊 Model Evaluation

After training, the model is evaluated using the testing dataset.

Two important metrics are used.

📈 R² Score (Coefficient of Determination)

R² Score measures how well the model explains the relationship between the input features and the target values.

Range

Typically from 0 to 1 (though it can be negative for very poor models).

1.0 → Perfect prediction.
0.9 → Excellent model.
0.7 → Good model.
0 → Model performs no better than predicting the average.

Higher R² means better performance.

Real-World Example

If R² = 0.95

The model explains about 95% of the variation in house prices.

 RMSE (Root Mean Squared Error)

RMSE measures the average prediction error in the original unit (Rupees).

Smaller RMSE means better predictions.

Real-World Example

If RMSE is:

Rs. 120,000

The model's predictions are usually about Rs. 120,000 away from the actual price.
